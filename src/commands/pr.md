# Создать PR

## Обзор
Создать хорошо структурированный pull request с правильным описанием, метками и ревьюерами.

## Шаги
1. **Подготовка ветки**
   - Убедиться, что все изменения закоммичены
   - Отправить ветку в удаленный репозиторий
   - Проверить, что ветка актуальна с main

2. **Написание описания PR**
   - Четко суммировать изменения
   - Включить контекст и мотивацию
   - Перечислить любые breaking changes
   - Добавить скриншоты при изменениях UI

3. **Настройка PR**
   - Создать PR с описательным заголовком
   - Добавить подходящие метки
   - Назначить ревьюеров
   - Связать связанные issues

## Шаблон PR
- [ ] Фича A
- [ ] Исправление бага B
- [ ] Unit тесты проходят
- [ ] Ручное тестирование завершено
